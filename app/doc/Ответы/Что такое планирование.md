## <u>Кратко</u>

Планирование (scheduling) — это процесс распределения ресурсов процессора между задачами или потоками для эффективного
выполнения программ.

## <u>Развернуто</u>

Планирование определяет порядок и время выполнения процессов или потоков в операционных системах и средах
программирования, обеспечивая многозадачность и оптимальное использование ресурсов.

1. **Основные задачи планирования**
    - Выбор следующей задачи или потока для выполнения.
    - Определение времени выполнения и приоритетов.
    - Управление состояниями задач (готовность, выполнение, ожидание).

2. **Типы планирования**
    - **Прерывное (Preemptive):** система может прервать текущий процесс ради другого с более высоким приоритетом.
    - **Не прерывное (Non-preemptive):** процесс выполняется до завершения или явного переключения.

3. **Планирование в Python**
    - Управляется планировщиком операционной системы.
    - В многопоточном Python управление потоками ограничено GIL — только один поток выполняется в один момент.
    - Асинхронное программирование использует событийный цикл для планирования заданий.

4. **Преимущества и недостатки**
    - **Преимущества:**
        - Позволяет многозадачности, улучшает отзывчивость и эффективность.
    - **Недостатки:**
        - Сложности в управлении и возможные состояния гонок.
        - Ограничения в Python (GIL) снижают эффективность многопоточности.

5. **Контекст использования**
    - ОС, многопоточные и асинхронные приложения, планирование задач и процессов.

6. **Пример (упрощённый):**
    ```python
    import threading
    import time

    def task(name):
        print(f"Задача {name} стартовала")
        time.sleep(1)
        print(f"Задача {name} завершена")

    t1 = threading.Thread(target=task, args=("A",))
    t2 = threading.Thread(target=task, args=("B",))

    t1.start()
    t2.start()

    t1.join()
    t2.join()
    ```

Таким образом, планирование — ключевой механизм управления выполнением задач, который распределяет ресурсы процессора
для оптимальной работы приложений и систем.
