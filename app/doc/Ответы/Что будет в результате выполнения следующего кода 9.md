## <u>Кратко</u>

В Python 3 вызов `len()` на объекте `map` приведёт к ошибке, потому что `map` возвращает итератор, у которого нет длины.

## <u>Развернуто</u>

Функция `map()` в Python 3 возвращает ленивый итератор, а не список, как это было в Python 2. Итератор не хранит все
элементы сразу и не поддерживает функцию `len()`. Поэтому попытка получить длину объекта `map` вызовет исключение
`TypeError`.

1. **Что происходит в коде**
    - `x = ['ab', 'cd']` — список из двух строк.
    - `map(list, x)` — создаётся итератор, который будет преобразовывать каждый элемент списка `x` (строки) в список
      символов, например `'ab'` → `['a', 'b']`.
    - `len(map(list, x))` — попытка получить длину итератора, что недопустимо.
    ```python
    x = ['ab', 'cd']
    print(len(map(list, x)))  # Ошибка: TypeError: object of type 'map' has no len()
    ```

2. **Как исправить**
    - Для получения длины результата нужно преобразовать итератор в список:
    ```python
    print(len(list(map(list, x))))  # Вывод: 2
    ```

3. **Преимущества и недостатки**
    - **Преимущества:**
        - Итераторы экономят память, не хранят сразу весь результат.
    - **Недостатки:**
        - Итераторы нельзя напрямую измерить с помощью `len()`.
        - Для операций, требующих длины, нужно явно преобразовывать их в списки.

4. **Контекст использования**
    - Во многих случаях функция `map()` используется для ленивой обработки данных.
    - Для операций, связанных с длиной или индексированием, результат необходимо приводить к списку.

Таким образом, в результате выполнения данного кода будет выброшена ошибка `TypeError`, так как невозможно применить
`len()` к объекту типа `map` в Python 3.
