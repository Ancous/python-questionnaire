## <u>Кратко</u>

Мы пишем `if __name__ == "__main__":` чтобы определить, выполняется ли скрипт напрямую, и выполнить код только в этом
случае, предотвращая запуск при импортировании модуля.

## <u>Развернуто</u>

В Python переменная `__name__` содержит имя текущего модуля. Если модуль запускается как основная программа, значение
`__name__` равно `"__main__"`. При импортировании в другой модуль — это имя модуля.

1. **Зачем используется `if __name__ == "__main__":`**
    - Позволяет разделить код, который должен выполняться только при запуске файла напрямую, и код, который должен быть
      доступен при импорте.
    - Удобно для написания тестов, примеров или отдельной логики запуска.

2. **Как это работает**
    - При прямом запуске Python-файла:
    ```python
    python script.py
    # переменная `__name__` в скрипте будет `"__main__"`.
    ```  
    - Если тот же файл импортируется:
    ```python
    import script
    # переменная `__name__` будет `"script"`, и блок внутри условия не выполнится.
    ```  

3. **Преимущества и недостатки**
    - **Преимущества:**
        - Предотвращает нежелательное выполнение кода при импорте.
        - Позволяет писать автономные модули с возможностью тестирования.
    - **Недостатки:**
        - При неправильном понимании может вызвать путаницу у новичков.

4. **Контекст использования**
    - В любом Python-файле, который может использоваться как скрипт и как модуль.
    - Для организации основных точек входа программы.

5. **Пример:**
    ```python
    def main():
        print("Программа запущена напрямую")

    if __name__ == "__main__":
        main()
    ```

Таким образом, конструкция `if __name__ == "__main__":` позволяет управлять выполнением кода в зависимости от способа
запуска Python-модуля.
